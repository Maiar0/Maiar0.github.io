Enhancement One Narrative

Code Repository

Enhancement One for the StockSense application focused on implementing modular and focused views, specifically the `DbSelectionView` and `SearchViewActivity`. The `DbSelectionView` allows users to select the database they want to interact with, while the `SearchViewActivity` provides an interface for viewing and searching items in the selected database. This modular approach adheres to the principle of separation of concerns, making the application more maintainable and scalable.

The inclusion of CSV import/export functionality significantly enhances the application's usability. Using the OpenCSV library, I implemented efficient data parsing and exporting, enabling seamless data transfer between StockSense and external tools like Excel. Additionally, integrating `RecyclerView` ensures a dynamic, user-friendly interface that scales efficiently.

Code Documentation and Maintainability

Throughout the enhancement, I prioritized clear documentation and maintainable code structures. Every function and variable has been thoroughly commented to ensure readability and future extensibility. Additionally, I created a workflow diagram to visually represent the application's design, further supporting future development efforts.

Error Handling and Security Improvements

Robust error handling mechanisms were integrated to ensure user-friendly feedback during CSV operations and database interactions. Additionally, I removed the password feature from this phase to simplify UI development while planning for a more secure authentication mechanism in future iterations.

Collaborative Features and Impact

The CSV import/export feature enhances collaboration by allowing teams to share inventory data between StockSense and external analysis tools like Excel. This aligns with the goal of supporting collaborative decision-making and efficient inventory management.

Technical Skills Demonstrated

This enhancement demonstrates a range of technical skills, including:

  • Android UI Design: Used `RecyclerView` to create a dynamic, user-friendly interface.
  • Data Handling: Implemented CSV import/export functionality using OpenCSV.
  • Error Handling & Security: Integrated structured exception handling to improve user feedback.
  • Version Control: Managed enhancements with Git branches to ensure structured development.

Challenges & Solutions

One of the primary challenges was debugging Supabase API interactions via Retrofit. Using Postman for API validation significantly improved debugging efficiency. Additionally, breaking down legacy code modularization into smaller tasks streamlined the refactoring process, ensuring a smooth transition to a more maintainable architecture.

Course Outcomes Addressed

  • Designing Computing Solutions: Developed a structured UI with modular components.
  • Applying Software Engineering Principles: Followed best practices in UI design and structured data management.
  • Professional Communication: Documented workflow diagrams and clear function-level commenting.
