StockSense: Lightweight Inventory Tracking

Originally developed in CS360: Mobile Architecture and Programming, StockSense began as a basic inventory tool for mobile devices. It utilized SQLite for local stock tracking but lacked multi-user support and real-time synchronization.

Evolution & Key Enhancements

  • Initial Limitations: No cloud access, no real-time collaboration, no authentication.
  • Major Improvements: Integrated cloud storage, secure login, and a smarter search system.
  • Scalability: Now supports multiple users managing inventory remotely.

Why It Matters

StockSense highlights my expertise in system design, security, and scalability. Taking an academic project and making it a real-world solution required mastering:

  • Cloud-based architecture for multi-user access.
  • Security best practices for authentication.
  • Performance optimization to handle larger datasets efficiently.
